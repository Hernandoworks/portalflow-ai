# Custom GPT Inspect Command v1

Command:

/inspect_custom_gpt

Purpose:
Create version-controlled governance for PortalFlow Custom GPT configuration.

## Inspection Scope

### 1. Identity Layer
Check:
- GPT name
- description
- role
- operating principles
- agent persona

### 2. Instruction Layer
Audit:
- system instructions
- reasoning workflow
- response standards
- safety rules
- approval matrix

### 3. Knowledge Layer
Inventory:
- uploaded knowledge files
- architecture docs
- API references
- product documentation

Output:
Knowledge Registry table:

| File | Purpose | Version | Owner | Status |

### 4. Action Layer
Inspect:
- OpenAPI schema
- available endpoints
- authentication
- action descriptions

Validate required PortalFlow actions:

GET /health
GET /commands
POST /plan
POST /execute

### 5. Connector Layer
Check integrations:
- Notion
- GitHub
- Google Drive

Validate:
- permissions
- scopes
- risk level

### 6. Agent Skills
Review installed skill modules:

- CTO Agent
- Engineer Agent
- Product Agent
- Security Agent
- Research Agent
- Knowledge Architect Agent

### 7. Registry Sync
Every GPT component must sync into:

- Agent Registry
- Prompt Registry
- Connector Registry
- Knowledge Registry
- Workflow Registry

## Output Format

Return:

## GPT Health Score

## Missing Components

## Security Risks

## Improvement Backlog

## Version Change Log


Version: 1.0
Owner: PortalFlow AI
